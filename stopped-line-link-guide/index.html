04

具体的な改善アクション

実際に直す順番にしています。ここだけ読めば作業に入れます。

1 正しい遷移先リンクを用意する

まず、どの LINE へ送るのが正しいかを確認し、その LINE へ送る short.io を用意します。既存の short.io が正しいと確認できた場合はそれを使い、新しく必要なら追加で発行します。

2 LINEリンクを直張りしている場合は short.io を活用する

UTAGE ページ内の LINE 直リンクを削除し、用意した short.io に差し替えます。LINE の直リンクは残しません。

3 差し替え後に実際に押して確認する

メールから UTAGE ページを開き、UTAGE ページから LINE へ進んで、意図した LINE に着地することを確認します。

このページで守る判断ルール
LINEリンクは必ず short.io で管理する。
ここに載せた short.io は参考情報であり、差し替え先だと断定しない。
停止中LINEへの直リンクは残さない。
差し替え後はメールから順に自分で押して確認する。

05

問題発見の背景と確認方法

まず今も送信待ちが残っているかを確認し、その後でメール本文、UTAGE ページ、LINE 着地先の順に辿って確認しました。

1. Mailchimp で対象メールを絞る

メールが停止されておらず、まだ送信待ちの人が残っているものだけを対象にしました。

2. 一時停止中メールは除外する

シナリオ自体が稼働中でも、メール単位で止まっているものは今回の修正対象から外しました。

3. メール内のリンク先を開く

画像、ボタン、テキストのどこから飛ぶのかを確認しながら、実際の UTAGE ページを開きました。

4. UTAGE ページ内の LINE リンクを見る

ページ内の LINE 直リンクと short.io の両方を確認し、どこに問題があるかを切り分けました。

5. 停止中LINEかどうかを照合する

Lステップの停止中一覧と照らし合わせて、停止中の「みかみ」に当たるかを確認しました。

6. 必要なものだけ残す

今も送信待ちが残っているメールだけを残し、古いシナリオや停止中メールは主対象から外しました。

自分で確認するときの手順
  1. Mailchimp で対象シナリオを開く
  2. 対象メールが停止されておらず、送信待ち人数が残っていることを確認する
  3. そのメール内の画像、ボタン、テキストのリンク先 URL を開く
  4. 開いた UTAGE ページの中で、ページ内に置かれている LINE リンクを確認する
  5. 同じページ内に short.io がある場合は、その short.io の向き先も確認する
  6. Lステップの停止中一覧で「みかみ」の利用停止日が 2024年9月17日であることを確認する

技術上は、Mailchimp の送信待ち人数が 0 より大きいかで確認しています。
